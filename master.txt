OVERVIEW
--------
Reap is a real estate analytics platform (app.getreap.ai) built as a React 19
single-page app deployed on GitHub Pages, backed by Supabase (Postgres, Auth,
Storage, Edge Functions). Organization: Suarez Global. Owner: Javier A. Suarez.

================================================================================
SESSION HISTORY
================================================================================

SESSION 1 — Foundation
- Built investor pipeline (CRUD for investors table)
- Investor detail view with tabs: overview, contacts, capital, linked deals,
  communications, documents, portal access
- Link/unlink deals to investors via pipe-delimited addresses
- Investor activities logging (calls, emails, meetings, notes)
- Portal email setup for investor portal access

SESSION 2 — Investor Portal & Updates
- Built InvestorPortalView (full portal for investors who log in)
- Investor updates system (post updates per deal)
- Update type categories: Construction Progress, Financial Update,
  Status Change, General Announcement (color-coded)
- Read receipt tracking (investor_update_reads table)
- Type filtering on portal updates
- Unread badges on portal nav tabs
- Public/Private toggle on updates (is_public boolean)
- Notify investors toggle with email/SMS recipient preview

SESSION 3 — Automated Notifications
- Supabase Edge Function (send-investor-notifications) for email + SMS
- Resend API integration for branded HTML emails
- Quo API integration for SMS notifications
- Notification preferences (email/SMS toggles) in portal settings
- Investor notification tracking (investor_notifications table)
- Branded email template: company name header, poster attribution,
  "Powered by Reap" footer
- Edge Function deployed via Supabase CLI
- All secrets configured: RESEND_API_KEY, QUO_API_KEY, FROM_EMAIL,
  FROM_NAME, QUO_FROM_NUMBER

SESSION 4 — Photos, Onboarding, Metrics, Funds, Routing
- Photo uploads on investor updates (multi-photo, Supabase Storage)
  - Photos stored in deal-documents bucket under update-photos/ prefix
  - Thumbnails with remove button on post form
  - Full-size display in admin previous updates + portal views
  - Cleanup on delete (removes from storage too)
- Investor onboarding flow (3-step welcome wizard)
  - Step 1: Welcome + feature overview grid
  - Step 2: Linked deals preview
  - Step 3: Notification preference setup
  - Only shows once (localStorage flag per investor)
  - Save prefs on completion, skip option available
- Engagement metrics dashboard (admin side)
  - Collapsible panel on Investor Updates tab
  - Summary cards: total updates, read receipts, sent/failed, delivery rate
  - Per-update performance breakdown (reads + sends)
  - Active investors list with read counts
- Fund-level grouping
  - New funds table (name, description, status, target_raise, capital_raised,
    target_irr, preferred_return, hold_period, vintage_year, deal_addresses)
  - Admin: Funds management panel in Investor Pipeline view
  - Admin: Create/edit fund modal with deal assignment picker
  - Portal: Funds tab with capital progress bar, metrics, deal list,
    recent updates, fund details footer
  - Funds tab dynamically shows/hides based on investor's linked deals
- Hash routing fixes for contacts/investors
  - #contacts/investors — lands on investors tab
  - #contacts/investor/{id} — deep links to specific investor
  - URL updates when navigating between contacts/investors tabs
  - URL updates when selecting an investor from list
  - Back button resets hash to #contacts/investors
  - Refresh now preserves investor tab and detail view

================================================================================
SUPABASE TABLES
================================================================================
deals                    — main deal/property records
deal_documents           — uploaded files (deal-documents storage bucket)
deal_activities          — activity timeline and notes
deal_shares              — deal sharing with permissions
organizations            — team/org management
org_members              — org membership
org_invites              — pending invitations
feature_flags            — per-org feature toggles
investors                — investor pipeline
  - portal_email         — email for portal access (Google OAuth)
  - contact_ids          — pipe-delimited (|||) contact IDs
  - linked_deal_addresses — pipe-delimited property addresses
  - notification_prefs   — JSON string {email:bool, sms:bool}
investor_updates         — posted updates per deal
  - is_public            — boolean (public=portal visible, private=internal)
  - photos               — JSON string of photo objects [{url, path, name}]
investor_update_reads    — read receipts (portal_email + update_id)
investor_notifications   — email/SMS delivery tracking (status, sent_at, error)
contacts / buyers        — contact management
portfolios               — portfolio groupings (deal_addresses pipe-delimited)
funds                    — investment funds (NEW — Session 4)
  - name, description, status, user_email
  - target_raise, capital_raised, target_irr, preferred_return
  - hold_period, vintage_year
  - deal_addresses       — pipe-delimited property addresses
market_intel             — market research data
user_profiles            — user account data

DEPLOYMENT
================================================================================
GitHub Pages via gh-pages package
Custom domain: app.getreap.ai (CNAME in build folder)
Deploy from /tmp to avoid EPERM issues on mounted folders:
  cd /tmp && rm -rf reap-deploy
  git clone https://presidentsuarez:<PAT>@github.com/presidentsuarez/reap-app.git reap-deploy
  cd reap-deploy && cp <workspace>/src/App.js src/App.js
  cp <workspace>/.env .env
  npm install && CI=false npm run build
  echo "app.getreap.ai" > build/CNAME
  npx gh-pages -d build

Edge Function deploy:
  supabase functions deploy send-investor-notifications
